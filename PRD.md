## 1. 프로덕트 한 줄 정의

**보드게임 카페를 배경으로, 누구나 매치를 열고 참여할 수 있는 O2O 매칭 플랫폼.**

---

## 2. 배경 & 목표

- 보드게임 카페에 혼자 가기 어렵거나, 같이 할 사람이 없는 문제를 해결
- 보드게임 카페와 협찬 파트너십 → 카페는 노출, 유저는 장소 제공
- **앱(iOS/Android) 메인** — 웹은 보조 (테스트/관리용)
- 출시 시점: 퀄리티가 충분하다고 판단될 때 (점진적 개선 방식)

---

## 3. 사용자

| 유형 | 설명 |
|------|------|
| **일반 유저** | 매치 탐색, 참여, 직접 개설 가능 |
| **호스트** | 매치를 만든 유저. 리워드 정산 받음 |
| **파트너 카페** | 협찬 보드게임 카페. 매치 장소 제공 |
| *(미래)* 운영진 | 공식 매치 개설 권한 |

---

## 4. 핵심 기능 정의

### 4-1. 매치 시스템 ✅ 구현됨
- 누구나 매치 생성 (게임 종류, 날짜, 장소, 인원 설정)
- 매치 탐색 (Discovery 피드)
- 참여 신청 / 취소 / 환불
- 호스트 취소 시 참여자 전원 환불 + 알림

### 4-2. 포인트 시스템 ⚠️ 부분 구현
- 현재: 숫자로만 구현 (가상 포인트), 서버 저장 완료
- **목표: 실제 현금 충전 방식** (PG사 연동 — 포트원/토스페이먼츠)
- 매치 참여 시 포인트 차감, 호스트에게 리워드 지급

### 4-3. 매너 온도 ✅ 구현됨
- 당근마켓 방식의 유저 온도 시스템
- 매치 후 별점(1~6) 제출 → 평균값이 매너 온도
- 리뷰 중복 방지, 매치 종료 후 30분 내 제출 가능

### 4-4. 채팅 ✅ 구현됨
- WebSocket 실시간 채팅 (매치별 채팅방, 참여자만 접근)
- REST 폴백 지원

### 4-5. 게임 도감 ✅ 구현됨
- **37종** 게임 등록 (장르별 탭 필터: 전략/파티/마피아/추리/카드/타일/고전/단어)
- 주요 프랜차이즈(레드버튼·히어로·체크메이트 등) 공통 인기작 분석 반영
- 게임 상세 (난이도, 룰 영상, 인원)

### 4-6. 보드게임 카페 파트너 🔜 미구현
- 카페 등록 → 매치 생성 시 카페 선택 가능
- 카페 프로필 페이지, 협찬 게임 목록 노출
- 추후 기획 구체화 필요

## 5. 기술 스택

| 영역 | 기술 |
|------|------|
| 앱/웹 프론트엔드 | Expo SDK 54 (React Native) — iOS/Android/Web 단일 코드 |
| 백엔드 API | FastAPI + SQLAlchemy |

| 마이그레이션 | Alembic |
| 테스트 | pytest (백엔드 24개) |

## 8. 알려진 기술 함정 (개발 시 주의)

> 자세한 내용은 `CLAUDE.md` 5번 섹션 참조

- `Alert.alert`는 웹에서 no-op → `dialog.js` 헬퍼 사용
- Alembic autogenerate가 `nullable` 반대 방향으로 뽑을 수 있음 → 생성 파일 검토 필수
- SQLite는 ALTER로 FK 추가 불가 → `batch_alter_table` 사용
- `seed.py`에 하드코딩된 `admin@boardway.io / admin123` 계정 → 배포 환경에 절대 시드 금지
- Expo SDK 핀 정합성 — `package.json`의 `expo` 버전과 다른 패키지 라인이 어긋나면 빌드 깨짐
- SQLite `:memory:` 테스트 — `StaticPool` 없으면 연결마다 새 DB 생성돼 "no such table" 발생
- Alembic 데이터 마이그레이션 — 컬럼 추가 후 기존 row 데이터는 별도 마이그레이션으로 UPDATE 필요
